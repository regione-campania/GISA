Cliccando sul tasto si apre la popup per gestire i dati aggiuntivi. Per le linee 'riproduzione animale' e 'igiene urbana' la schermata si presente come segue:

Se i dati sono presenti viene popolata la tabella 'Elementi già presenti' altrimenti viene mostrata la form di inserimento. Se i dati sono presenti viene mostrata anche la tabella "MODIFICA DETTAGLI LINEA DI ATTIVITA'" in cui inserire i dati aggiornati. I campi 'codice stazione' e 'codice recapito' di stabilimenti inseriti prima della modifica sul codice stazione non sono soggetti a controlli di formattazione. Il codice stazione e il codice recapito possono essere modificati solo se non presenti. Se già inseriti è possibile effettuare la modifica tramite segnalazione all'help desk. La differenza sta nella possibilità di poter inserire più targhe ed eliminarne una tramite il tasto 'Elimina' infatti, come specificato nella schermata, per l'aggiornamento del singolo automezzo bisogna eliminare e reinserire il dato.
Dismissione
Tramite il bottone "Dismetti" e' possibile procedere alla dismissione "rapida" dei dati che prevedono una data dismissione (Es. automezzi) senza dover reinserire il dato aggiornato.
La funzionalita' puo' essere utilizzata anche per la modifica di un dato (Dismissione -> Reinserimento).
Cliccando sul bottone "Dismetti", il sistema obblighera' ad inserire le note e la data. Il dato sara' dismesso in modo da poter essere reinserito.
Questa operazione sara' tracciata.


Per le linee di COMMERCIO ALIMENTI USO UMANO è possibile salvare una lista di distributori mediante il caricamento di un file.

Il file da caricare dovrà essere in formato txt o csv separato dalcarattere ";" e dovra' contenere i seguenti campi:
  • matricola: campo di testo
  • data Inizio: campo di testo formato dd/MM/yyyy
  • Ubicazione: campo di testo
  • indirizzo: campo di testo
  • comune: campo di testo
  • cap: campo di testo
Dopo aver scelto il file cliccando sul pulsante salva il sistema eseguira' la seguente elaborazione:
  • Controllo validita' del file : se il file non è corretto viene segnalato all'utente
  • Controllo Validita' del record : se i dati di un record non sono corretti, il record viene scartato
  • Controllo Esistenza del record(per Matricola) : se il record non esiste viene inserito correttamente
  • Controllo Esistenza del record: se il record esiste viene aggiornato se la data del distributore presente nel sistema è antecedente a quella letta dal file.
  • Al termine dell'elaborazione viene ricaricata la pagina.

Il pulsante lista file caricati mostra la lista degli import eseguiti

La lista mostra la data in cui è avvenuto il caricamento, l'esito dell'elaborazione e la possibilità di scaricare la lista di record elaboati indicando per ognuno l'esito dell'elaborazione

La presente gestione viene visualizzata anche dagli utenti trasportatori che accedono dal portale esterno.
